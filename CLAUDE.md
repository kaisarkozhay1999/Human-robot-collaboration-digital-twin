# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

SmartLab is a **Mixed Reality (MR) laboratory control and monitoring application** built in Unity 2022.3 LTS, targeting HoloLens 2 (UWP/IL2CPP). It creates a digital twin interface for controlling physical lab equipment (robotic arms, smart plugs, lighting) and monitoring environmental sensors via MQTT.

## Build & Development

- **Engine:** Unity 2022.3.62f3 LTS — open and build through the Unity Editor
- **Solution:** `SmartLab 3.5.2026.sln` / `Assembly-CSharp.csproj` (auto-generated by Unity)
- **Target:** Universal Windows Platform (UWP), IL2CPP scripting backend
- **Primary Scene:** `Assets/Scenes/SampleScene.unity` — the only active scene, contains all controllers and UI
- **MCP Integration:** UnityMCP is configured in `.claude/settings.local.json` — use `find_gameobjects`, `manage_scene`, `manage_gameobject` tools to inspect and modify the running Unity Editor

## Architecture

### Data Flow

```
MQTT Broker (192.168.0.190:1883)
  ├─ lab/sensors/all              → RoomTelemetrySubscriber   → Environmental UI
  ├─ lab/room1/socket*/telemetry  → SocketTelemetrySubscriber → Power UI
  ├─ robot/state                  → ER9ProFullController       → Robot pose
  ├─ robot/control                → ER9ProFullController       → Robot commands
  └─ lab/room1/bulb1/cmd         → MQTTBulbController         → Lighting

UDP (Port 5005)
  └─ StopGoReceiver → republishes to MQTT (robot/control)

HTTP (192.168.0.190:8080)
  └─ MJPEGViewer → RawImage texture (IP camera feed)
```

### Threading Model

MQTT callbacks fire on background threads. All Unity API calls (UI updates, transform changes) must be marshalled to the main thread via **`UnityMainThreadDispatcher`** (singleton queue). `SocketTelemetrySubscriber` and `RoomTelemetrySubscriber` use `lock` for thread-safe field access.

### Key Systems

| Script | Purpose |
|--------|---------|
| `ER9ProFullController.cs` | 6-DOF robot arm (ER9 Pro) — receives joint angles via MQTT, drives pose via Lerp smoothing with calibration offsets |
| `StressAnalysisController.cs` | Real-time joint stress heatmap — static torque model, Green→Yellow→Red coloring, pulse animation >75% rated torque. Accesses ER9Pro private fields via **reflection** |
| `MJPEGViewer.cs` | MJPEG stream decoder — parses FF D8/FF D9 JPEG markers, pre-allocated buffers to minimize GC, background network thread |
| `RoomTelemetrySubscriber.cs` | Environmental telemetry (temp, humidity, pressure, light, motion) — JSON via Newtonsoft.Json |
| `SocketTelemetrySubscriber.cs` | Per-socket power metering (watts, volts, amps) — same JSON pattern |
| `StopGoReceiver.cs` | Dual-protocol command router: UDP port 5005 + MQTT |
| `MQTTBulbController` (`Bulb 1.cs`) | Smart bulb control (brightness 0–255, RGB) |
| `UnityMainThreadDispatcher.cs` | Thread-safe action queue; required for any background→UI updates |
| `DeviceUIController.cs` | Toggles device detail panels in the MR UI |

### Technology Stack

- **Networking:** M2MQTT (Eclipse Paho, modified for UWP) — located in `Assets/M2MqttUnity-master/`
- **XR:** MRTK 4.x + OpenXR 1.14.3 + XR Hands 1.4.0 (hand tracking)
- **UI:** TextMeshPro 3.0.7, UGUI (for RawImage video display)
- **Serialization:** Newtonsoft.Json 3.2.1
- **Input:** Unity Input System 1.14.0

### Important Implementation Notes

- **Hard-coded network config:** Broker IP (`192.168.0.190`), camera URL, and MQTT topics are literals in source files — not externalized to ScriptableObjects
- **Reflection usage:** `StressAnalysisController` uses `System.Reflection` to read private joint angle fields from `ER9ProFullController` — changing field names/visibility there will break stress analysis silently
- **MRTK auto-generated config** lives in `Assets/MRTK.Generated/` — do not hand-edit
- **IL2CPP build artifacts** in `Build/` and `Library/` are generated — do not commit
